Mission 2 – Bonnes pratiques & sécurité des données
Objectif : Identifier les mauvaises pratiques et proposer des mesures correctives
← Portail
Durée estimée : 20 min

Scénario

Vous accédez à un intranet fictif d’entreprise. Plusieurs éléments techniques et organisationnels posent problème au regard du RGPD et de la sécurité des données.

Extrait de code – Authentification

const db = {
  adminUser: "admin",
  adminPassword: "Admin123!",
  backupPassword: "Backup2024!",
};

function login(user, pass) {
  if (user === db.adminUser && pass === db.adminPassword) {
    console.log("Connexion réussie, rôle : ADMIN");
    // TODO: ajouter des logs plus détaillés avec IP, mot de passe, etc.
  }
}

➤ Mots de passe en clair dans le code, réutilisation probable, absence de gestion de rôles fine.

Dossier accessible publiquement

Un dossier /backup/ est accessible depuis le navigateur : 

/backup/users_full_export.csv
/backup/hr_salaries_2023.xlsx
/backup/it_admin_passwords.txt

➤ Sauvegardes non protégées, données potentiellement sensibles exposées.

Votre mission

  • Listez toutes les mauvaises pratiques visibles dans ces extraits.
  • Proposez des mesures correctives (techniques et organisationnelles).
  • Reliez ces mauvaises pratiques aux obligations RGPD (sécurité, confidentialité, intégrité).
  • Identifiez les risques pour les personnes concernées (exposition de données, usurpation, etc.).

À la fin de la mission, vous devez être capable de :

  • Expliquer pourquoi la sécurité est un pilier du RGPD.
  • Donner des exemples concrets de mesures de sécurité adaptées.
  • Relier les risques techniques à des impacts humains et juridiques.
Mission 2 – Environnement fictif. Les exemples sont volontairement caricaturaux pour faciliter l’analyse.
